# Garret widget theme

Garret serves a shared stylesheet to **every** widget on the widget's own origin, so you get the
native macOS look for one line — no CSS to write, nothing to bundle:

```html
<link rel="stylesheet" href="~theme.css" />
```

`~theme.css` is generated by the app and served same-origin, so it passes the strict widget CSP
(`style-src 'self'`). Use the classes and tokens below; the app owns the styling, so your widget
stays visually consistent as Garret evolves.

## Design tokens (CSS variables on `:root`)

| Token | Use |
| --- | --- |
| `--text` / `--text-2` / `--text-3` | primary / secondary / tertiary text |
| `--accent` | system blue (links, active states) |
| `--danger` | errors |
| `--surface-input` | control fill (switch track, pills) |
| `--surface-hover` | row hover |
| `--hairline` | 0.5px dividers |
| `--r-input` / `--r-group` | input / grouped-container radius |
| `--font` | system font stack (already applied to `body`) |

The theme sets `body` background, color, font, and scrollbars — so an empty widget already looks
native.

## Layout / chrome

- `.g-bar` — top toolbar (draggable). Put the title + buttons here.
  - `.g-title` — the widget title (ellipsized).
  - `.g-icon-btn` — a 26px icon button (e.g. ⚙ / ↻); add `.on` when active.

## Lists

- `.ticket-list` — vertical list container (scrolls).
- `.list-caption` — small uppercase section label.
- `.ticket` — a row button. Children: `.ticket-dot` (+ state), `.ticket-key`, `.ticket-summary`,
  `.ticket-status` or a `.status-pill`.
- `.status-pill` — colored pill. State classes: `.todo`, `.progress`, `.open`, `.done`, `.merged`,
  `.declined`.
- `.ticket-dot` — same state classes as the pill.

## Pull-request list

- `.pr-widget` — container. `.pr-group-head` (`.pr-group-name` + `.pr-group-count`), then
  `.pr-row` (`.pr-row-title` + `.pr-meta`).
- `.pr-meta` children: `.pr-author`, `.pr-reviewers` (with `.pr-rev-dot` + `.approved` /
  `.changes_requested` / `.pending`), `.pr-comments`, `.status-pill`.
- `.pr-unmute` — a subtle full-width footer button.

## Settings / config form

Grouped rows, System-Settings style:

- `.settings-form` → `.settings-item` → `.settings-group` → `.settings-row`.
- `.settings-row` children: `.settings-row-label` + `.settings-row-control`.
- Controls: `.row-input` (text/number), `.row-select` (styled `<select>`),
  `.switch` (button with a `.switch-knob` child; add `.on` when toggled).
- `.settings-section-label` — a bold label above a group. `.settings-note` — fine print.

## Empty / error states

- `.svc-empty` — centered muted message (supports `<b>` emphasis).
- `.svc-error` — an error message (in `--danger`).

See the `atlassian` pack in this repo for a full working example.
